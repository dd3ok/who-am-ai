# Implement — Google GenAI project-id 설정 *(Deprecated: see /docs/impl/251107-170714-genai-apikey-설정.md)*

## 변경 요약
- `spring.ai.google.genai` 섹션에 `project-id`/`location`을 환경 변수 기반으로 추가했다.
- 임베딩 전용 네임스페이스에도 동일한 값을 주입해 Spring AI VectorStore 자동 구성에서 요구하는 연결 정보를 충족시켰다.

## Why / Scope
- Why: 로그에서 `Google GenAI project-id must be set!` 예외가 발생해 전체 컨텍스트 초기화가 중단되었고, 설정 누락을 보완해야 했다.
- Scope: `application.yml` 내 Google GenAI 관련 속성만 수정하며, 다른 모듈/로직은 변경하지 않았다.
- 참조: Spring AI Google GenAI 설정 가이드 (see: /spring-projects/spring-ai).

## Plan 대비 변경 사항 (Drift)
- 없음. Plan 문서(`/docs/plan+design/251107-165938-genai-프로젝트id-점검.md`)에서 정의한 범위와 동일하게 진행했다.

## 코드 주요 포인트
- `spring.ai.google.genai.project-id`와 `spring.ai.google.genai.location`을 각각 `GEMINI_PROJECT_ID`, `GEMINI_LOCATION`으로 매핑.
- `spring.ai.google.genai.embedding.project-id/location`도 명시하여 `GoogleGenAiEmbeddingConnectionDetails`가 필요한 변수를 모두 전달.

## 배치/운영 플로우
- 배치 구성 변경 없음. Spring Boot 애플리케이션 기동 시 자동 설정이 추가 파라미터를 받아 초기화된다.

## 테스트 시나리오 & 예상 결과 (실행하지 않음)
- 시나리오 1: `GEMINI_PROJECT_ID`, `GEMINI_LOCATION` 환경 변수 설정 후 애플리케이션 기동 → Spring Context 초기화 성공, VectorStore 빈 생성 완료.
- 시나리오 2: `GEMINI_PROJECT_ID`가 누락된 상태에서 기동 → 동일 예외 재현, 로깅을 통해 환경 변수 미설정을 감지.
- 엣지 케이스: `GEMINI_LOCATION` 미설정 시 기본값 `us-central1`을 사용하여 정상 기동.
